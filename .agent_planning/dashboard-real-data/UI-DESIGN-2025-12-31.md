# Turbo-V2 Experiment Dashboard - UI Design

## Design Goals

1. **Live Monitoring** - Watch experiments execute in real-time
2. **Historical Browsing** - Search, filter, analyze past experiments
3. **Parallel Execution** - Run multiple experiments, monitor all at once
4. **Control** - Cancel, pause, retry experiments
5. **Configuration** - View and tweak passes before and during runs

## Interaction Patterns

### Running Multiple Experiments
- Click "+ New Experiment" opens form without leaving current view
- "Create & Run" starts immediately, redirects to Live Monitor
- "Create" saves as pending, stays on create form for batch creation
- Live Monitor shows ALL running experiments simultaneously

### Cancelling an Experiment
- Click "Cancel" on running experiment
- Confirmation dialog: "Cancel experiment? Progress will be lost."
- On confirm: POST /api/experiments/:id/cancel
- Server aborts executeTurboV2, updates status to "cancelled"
- Partial results kept in storage (for debugging)

### Tweaking Passes Mid-Run
- NOT SUPPORTED (would require checkpoint modification)
- Instead: Cancel current, create new with modified passes
- Future: "Clone & Modify" button on running experiment
